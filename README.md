# MAX Bot — платформа продажи онлайн-курсов

Бот для мессенджера **MAX** (VK Teams), реализующий полный цикл продажи подписок на онлайн-курсы: от каталога и оплаты до автоматического управления доступом к закрытым каналам.

---

## Стек технологий

- **Python 3.10+** — asyncio, async/await
- **aiohttp** — асинхронный HTTP-клиент и webhook-сервер
- **asyncpg** — асинхронный драйвер PostgreSQL
- **PostgreSQL** — основная база данных
- **openpyxl** — экспорт отчётов в Excel

---

## Функциональность

### Пользователь

- Каталог курсов с описанием, ценой и медиа (фото/видео)
- **Два типа курсов:**
  - **Марафон (фиксированная дата)** — доступ до конкретной даты и времени, одинаковой для всех покупателей
  - **Самостоятельный (фиксированная длительность)** — доступ N дней с момента покупки, индивидуально для каждого
- Применение промокодов со скидкой при покупке
- Поддержка нескольких платёжных систем на выбор
- Автоматическое получение инвайт-ссылок на закрытые каналы после оплаты
- Личный кабинет: список активных подписок с датами окончания на русском языке
- Бонусный раздел: автоматическая выдача файлов (гайды, материалы) по активным подпискам
- Форма обратной связи с пересылкой сообщений (текст + вложения) администратору
- Уведомления об истечении подписки (за 3 дня, за 1 день, в день окончания)

### Администратор

**Управление тарифами:**
- Создание, редактирование, удаление тарифов
- Настройка цен: базовая, старая (зачёркнутая), при продлении, при продлении активной подписки
- Категории тарифов с произвольной сортировкой
- Привязка нескольких каналов/чатов к одному тарифу
- **Тип курса задаётся через настройки:**
  - Установить `end_date` → курс становится марафоном; поля продолжительности сбрасываются автоматически
  - Установить `duration_days` → курс становится самостоятельным; `end_date` сбрасывается автоматически
- Ограничения: лимит активаций, дата начала продаж, доступные группы пользователей
- Подарочные файлы: прикрепление документов к тарифам для автовыдачи

**Промокоды:**
- Скидка в процентах
- Ограничения: количество активаций, количество использований на пользователя, срок действия
- Привязка промокода к конкретным тарифам или пользователям

**Платёжные методы:**
- Динамическое добавление/удаление провайдеров через интерфейс
- Настройка реквизитов (shop_id, secret_key) без правки кода

**Пользователи:**
- Блокировка/разблокировка
- Ручная выдача и отзыв подписки
- Перенос подписки между пользователями
- Лог действий пользователя

**Рассылки:**
- Всем пользователям
- Только платящим / только неплатящим
- Подписчикам конкретного тарифа

**Статистика и отчёты:**
- Общее число пользователей и прирост за 30 дней
- Активные подписки и выручка за период
- Экспорт данных в Excel

**Настройки бота:**
- Все пользовательские тексты редактируются через интерфейс без деплоя (включая тексты после активации и ссылки на ресурсы)

## Платёжные системы

| Провайдер | Способ интеграции |
|---|---|
| **ЮКасса** | REST API, polling статуса |
| **Prodamus** | Payform API, webhook + polling |
| **Альфа-Банк** | REST API (Sber-совместимый), polling статуса |

Webhook-сервер для уведомлений о платежах поднимается параллельно с ботом на настраиваемом порту.

## Архитектура

```
bot.py                  # Точка входа, polling-цикл, фоновые задачи
├── handlers.py         # Пользовательские сценарии (FSM, оплата, подписки)
├── admin_handlers.py   # Административная панель
├── database.py         # Все запросы к PostgreSQL (asyncpg)
├── payments.py         # Провайдеры оплаты (стратегия)
├── max_client.py       # Клиент MAX Bot API
├── keyboards.py        # Клавиатуры пользователя
├── admin_keyboards.py  # Клавиатуры администратора
├── fsm.py              # In-memory FSM для многошаговых сценариев
├── config.py           # Конфигурация (токен, БД, вебхук) — не в git
└── config.example.py   # Шаблон конфигурации
```

**Ключевые паттерны:**
- **FSM (конечный автомат)** — управление многошаговыми диалогами (создание тарифа, ввод промокода и т.д.)
- **Strategy** — абстрактный базовый класс `PaymentProvider`, отдельные реализации под каждый провайдер
- **Polling + Webhook гибрид** — long-polling от MAX API для сообщений, aiohttp-сервер для платёжных уведомлений
- **Фоновые задачи** — два параллельных asyncio-цикла: проверка истёкших подписок (каждые 60 сек) и проверка статуса платежей (каждые 15 сек)

## Схема базы данных

| Таблица | Назначение |
|---|---|
| `users` | Пользователи бота |
| `tariffs` | Курсы/тарифы с настройками цен и доступа |
| `categories` | Группировка тарифов |
| `tariff_resources` | Каналы/чаты, привязанные к тарифу |
| `purchases` | История покупок и статусы подписок |
| `payment_methods` | Настройки платёжных провайдеров |
| `promo_codes` | Промокоды и условия их применения |
| `promo_activations` | Использования промокодов |
| `gift_files` | Бонусные файлы для автовыдачи |
| `gift_file_tariffs` | Связь файлов с тарифами |
| `bot_texts` | Редактируемые тексты бота |
| `user_logs` | Лог действий пользователей |

## Установка

**1. Клонировать репозиторий и установить зависимости:**
```bash
pip install -r requirements.txt
```

**2. Настроить конфигурацию:**
```bash
cp config.example.py config.py
# Заполнить config.py своими значениями
```

**3. Создать базу данных PostgreSQL:**
```bash
psql -U postgres -c "CREATE DATABASE dbname;"
# Инициализация таблиц происходит автоматически при первом запуске
```

**4. Запустить бота:**
```bash
python bot.py
```

## Конфигурация

Все параметры задаются в `config.py` (шаблон — `config.example.py`):

| Параметр | Описание |
|---|---|
| `BOT_TOKEN` | Токен бота в MAX |
| `ADMIN_IDS` | Set из user_id администраторов |
| `DATABASE_URL` | Строка подключения к PostgreSQL |
| `WEBHOOK_HOST` | Хост для webhook-сервера |
| `WEBHOOK_PORT` | Порт для webhook-сервера |
| `WEBHOOK_BASE_URL` | Публичный URL для приёма платёжных уведомлений |
| `OFERTA_URL` | Ссылка на договор оферты |
| `PRIVACY_URL` | Ссылка на политику конфиденциальности |
